# SEO 落地执行 SOP

结合哥飞经验与项目框架要求，整理一份可直接执行的 SEO SOP，按阶段拆解动作与检查点。

## 0. 准备与定位

- 关键词即品牌：核心词注册独立域名，域名中包含主关键词；目标是让该词成为品牌词，争取 sitelinks。
- 单词单域/单词单页：高意图词单独建站或最少独立落地页，避免在老站深层内页稀释权重。
- 技术基线：全站 SSR、可控 TDK、h 标签层级、OG/社媒信息、多语言 + hreflang、CDN/压缩/响应式等（docs/frameworkSeo.md）。

## 1. 关键词获取与筛选

- 渠道：Similarweb「关键词研究-着陆页」找近期新点击量词；结合 Google Trends 验证走势；补充 Ahrefs/ Semrush、Google 搜索 `intitle:"best websites"`、YouTube/GitHub/游戏站等长尾来源。
- 竞争评估：查看前 10 SERP 是否品牌词、首页 vs 内页、外链域名数、域龄、流量；品牌词+sitelinks 直接放弃，低竞争长尾优先。
- 批量清单：为每个词记录搜索趋势、KD、主 SERP 页面类型、预期变现（工具/广告/注册）。

## 2. 页面模型与内容生成

- 精品工具页公式：工具页 + 落地页 + 结果展示三合一，同页既有表单/交互，又有足量文案与结果列表，提升停留时长与行为数据。
- 结构要求：
  - TDK：`getMetadata` 动态生成，主关键词前置，描述含 CTA，keywords 控制密度 ~3%。
  - 内容：要求在1000字左右
  - h 标签：H1 唯一（主词），H2/H3 分层覆盖同义词/问题词。
  - 文案：SSR 输出，不依赖前端异步；多语言版本写入 locale JSON，自动注入 hreflang。
  - 交互：工具表单可直接提交/展示结果；结果列表可预置精选示例，保证首屏有可视内容。
  - 社媒：补齐 `og:title/description/image`；首选能代表结果页的图片。
  - 技术性能：开启静态资源压缩、缓存与图片懒加载，Core Web Vitals 过线。

## 3. 站点搭建与提交

- 新站上线后在 GSC 网址检测提交索引；生成/更新 sitemap.xml 并确保新增子页面自动写入；robots.txt 允许主路径抓取。
- 多语言：主域承载所有语言，避免分流到子域；使用 hreflang 指向对应 locale 页面。
- GEO/AI 收录：可视情况补充 llms.txt，暴露站点地图与主要功能页入口。

## 4. 外链与推广

- 初期外链：搜索 “blog commenting sites list” 建外链池，挑同行业且评论不卷的页面发外链；老站互挂友链（底部链接位）。
- 广告试投：按站点/语言拆 Campaign，日预算 50-100 USD；先按点击跑量积累画像，跑通后转注册/转化；跑得好的 AdGroup 拆成独立 Campaign。

## 5. 数据驱动优化

- GSC 监控：筛选展示高但排名靠后的词，提升关键词密度或补充内页；观察点击/跳出/停留，迭代文案与交互。
- 内页扩展：基于新词清单与 GSC 热点，持续新增长尾子页面；保持博客更新节奏（产品/热点如 “what-is-xxx”）。
- 行为提升：在落地页合理加入 iframe/工具交互，增加停留时长，降低跳出。

## 6. 运维节奏（示例 30-60 天）

- 第 1 周：确定主关键词与域名，完成首个精品工具页（多语言、OG、TDK、sitemap、hreflang）。
- 第 2-3 周：批量上 5-10 个低竞争长尾页；提交索引；建立初始外链；跑小额 Ads 测试。
- 第 4-6 周：依据 GSC 数据调密度/文案/结构；新增长尾页；优化 Core Web Vitals；扩展博客 1-2 篇/周。

## 7. 检查清单

- TDK 动态生成且关键词前置；H1 唯一，H2/H3 层级清晰，关键词密度约 3%，内容要求在1000字。
- SSR 输出完整内容；OG/社媒、sitemap、robots、hreflang 全量可用。
- 页面含可用工具交互 + 结果展示，首屏可视；多语言内容已落地到 locale JSON。
- GSC 已提交并有数据；外链与投放计划已执行；博客/内页更新节奏稳定。
